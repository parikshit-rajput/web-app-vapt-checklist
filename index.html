How to Use This Checklist 0 items

• Use each row as a direct test action during authorized Web/API VAPT or bug bounty testing.

• Each checklist item states what to perform, examples or payloads to try, and the expected secure behavior.

• Mark an item as checked only after testing the web UI and the corresponding API request where applicable.

• Do not perform destructive tests outside approved scope. Use safe payloads and controlled test accounts.

Information Gathering 0 items
Search Engine Discovery & Information Leakage 3 items

  • Use search engines to look for sensitive indexed content under the target domain.

  • Check Wayback/cached pages for old endpoints, parameters, backup files, and removed sensitive content.

  • Search GitHub/GitLab/public docs for organization name, domain, API keys, internal URLs, credentials, or environment files.

Fingerprint Web Server 3 items 
  • Capture responses and check headers/banner/version from web server or proxy.

  • Review HTTP headers for technology disclosure.

  • Use safe fingerprinting to identify OS/service details only within scope.
Review Webserver Metafiles 3 items

  • Open /robots.txt and note disallowed paths, admin paths, private endpoints, or test directories.

  • Open /sitemap.xml and check for hidden, old, staging, or sensitive pages.

  • Check /.well-known/security.txt and other .well-known files for useful security/contact metadata or hidden configuration.

Enumerate Applications on Webserver 6 items

  • Use subdomain tools and search to discover all subdomains, then verify which are live and what they expose.

  • Fuzz the Host header against the target IP/domain to find hidden apps hosted on the same server.

  • Perform directory/path enumeration for unlinked apps and old panels.

  • Search common admin paths and verify whether they are protected with auth, MFA, rate limit, and access control.

  • Capture API traffic and check common API documentation/version paths.

  • Search for non-production environments and verify whether they expose weaker controls or real data.

Review Comments & Metadata 6 items

  • Open page source and Burp responses to find comments exposing internal notes, hidden paths, credentials, test data, or disabled functionality.

  • Review JS files for TODO/FIXME notes, API endpoints, feature flags, environment names, or internal logic.

  • Send invalid/malformed requests and inspect responses/JS/API errors for stack traces, file paths, DB errors, or debug logs.

  • Search JS/config/API responses for API keys, tokens, secrets, client IDs, environment variables, or third-party credentials.

  • Download public PDFs/DOCX/XLSX/images and inspect metadata for author names, internal usernames, software versions, or file paths.

  • Look for exposed .map files and verify whether they reveal original source code, hidden routes, comments, or sensitive logic.

Identify Application Entry Points 3 items

  • List every place where the user can submit or control data.

  • Record endpoint path, method, parameters, headers, content type, auth requirement, and role requirement for each API.

  • List actions that change state or expose sensitive data.

Map Execution Paths 3 items

  • Capture full step sequence for key flows and note required preconditions for each step.

  • Record request order and dependencies for important flows.

  • Try calling later APIs directly without completing previous steps.

Fingerprint Web Application 5 items

  • Login with each available role and compare visible pages, buttons, APIs, and actions.

  • Document business workflows step-by-step so they can be tested for skip/replay/repeat/tamper abuse.

  • Group APIs by functionality and record path, method, auth, parameters, and response.

  • Send invalid, missing, expired, malformed, and unauthorized requests to observe error style and leakage.

  • Record all protections in use: CAPTCHA, MFA, OTP, CSRF, rate limits, JWT/session validation, secure cookies, CORS, WAF/CDN, and re-authentication.

Map Application Architecture 6 items

  • Observe how frontend calls backend APIs, including base URL, request format, auth method, and common headers.

  • Check whether APIs pass through a gateway/proxy by observing common domains, routing behavior, headers, rate limits, and error patterns.

  • Determine whether auth uses JWT, session cookies, OAuth/SSO, API keys, OTP, or MFA, and where tokens are stored.

  • Group APIs by backend service/functionality and compare security behavior across services.

  • List external services loaded or called by the app and check whether sensitive data/tokens/callbacks are exposed.

  • Check DNS/headers/cache behavior to identify CDN, WAF, reverse proxy, or load balancer.

Configuration & Deployment Management Testing 0 items
Test Network / Infrastructure Configuration 7 items 
  • Scan target host/IP and verify only required ports are publicly exposed.

  • Identify services behind open ports and check if risky services are publicly exposed.

  • Verify firewall/security group restrictions for admin/internal services.

  • Inspect responses, headers, JS, APIs, and errors for private IPs/hostnames/server names.

  • Find cloud storage URLs/buckets referenced by app and verify read/write/list permissions.

  • Check caching behavior, origin exposure, and whether private pages are cached.

  • Test TLS protocols, certificate validity, and weak ciphers.

Test Application Platform Configuration 5 items

  • Look for debug mode, test mode, dev banners, debug endpoints, or verbose logs in app/API.

  • Trigger invalid requests and verify errors do not expose stack traces, DB errors, internal paths, or framework details.

  • Look for default pages, install/setup pages, default credentials, sample apps, or exposed management consoles.

  • Identify server/framework/CMS/library versions and compare with known vulnerable versions.

  • Review security headers and cookie flags.

Test File Extension Handling 3 items 
  • Directly request sensitive file extensions and common config/backup names.

  • Request/upload files and verify the server does not rely only on extension or declared Content-Type.

  • Open uploaded/static risky files and check whether they execute/render inline.
Backup & Unreferenced Files 4 items

  • Bruteforce common backup names and extensions.

  • Look for temporary/editor-generated files.
  • Check old/deprecated directories or API versions.

  • Perform path enumeration for unlinked functionality.

Enumerate Admin Interfaces 3 items 
  • Check common admin/management URLs.
  • Check admin API paths and test as unauthenticated/low-privilege user.

  • Verify MFA, rate limit, session timeout, secure cookies, no default credentials.

Test HTTP Methods 4 items

  • Send OPTIONS and manual requests to identify allowed methods.

  • Test if PUT/DELETE/TRACE allow upload, modification, deletion, or trace reflection.

  • Send same request with method override headers/params to see if backend treats it as a different method.

  • Send the same restricted endpoint with GET/POST/PUT/PATCH/DELETE and verify authz is enforced for each.

Test HTTP Strict Transport Security 4 items

  • Verify Strict-Transport-Security exists on HTTPS responses.

  • Verify max-age is strong and not too short.

  • Verify includeSubDomains only if all subdomains support HTTPS.

  • Access HTTP URL and verify secure redirect without leaking cookies/tokens.
Test RIA Cross Domain Policy 3 items 
  • Open root/application paths for Flash policy file.
  • Open root/application paths for Silverlight policy file.

  • Review policy files for wildcard domains or unrestricted headers.

Identity Management Testing 0 items
Role, Registration and Account Lifecycle 9 items

  • Create/use each role and compare allowed pages, APIs, menu items, actions, and data access.

  • Register with email/phone and try using account without completing verification.

  • Register with same email/phone using case changes, spaces, plus addressing, Unicode, or alternate formats.

  • Add hidden fields in registration body.

  • Check newly created accounts for default roles, feature access, and API permissions.

  • Compare valid/invalid usernames across login/signup/forgot password.
  • Try reserved/system usernames and confusing formats.
  • Use guest/demo/training accounts and check restricted pages/APIs/data.

  • Suspend/disable an account and try login/API/session reuse; then test reactivation flow.

Authentication Testing 0 items
Login and Authentication Controls 10 items

  • Submit login and verify credentials/tokens never travel over HTTP or URL query string.

  • Try approved/common default credentials on login/admin/dev panels.
  • Send repeated login attempts for same and different users/IPs.

  • Access authenticated pages/APIs without token/cookie or with modified/expired token.

  • Search old login paths and verify they do not bypass current MFA/CAPTCHA/lockout/security controls.

  • Enable remember-me and inspect token/cookie lifetime, storage, rotation, and invalidation after logout/password reset.

  • Login, view sensitive page, logout, then use browser back/cache/offline mode.
  • Try weak/common/passwords based on username/email.

  • Try guessable answers, repeated guesses, and public-info answers.

  • Compare web, mobile, API, old endpoints, and SSO flows for inconsistent authentication controls.

2FA / MFA / OTP Bypass Techniques 12 items

  • Intercept 2FA verification response and check whether client-side changes can unlock account.

  • Trigger 2FA/OTP and inspect response, logs, JS, and API body for OTP/code exposure.

  • Use a valid 2FA code once, then submit it again or from another session/device.

  • Attempt many wrong OTP values and observe retry limits, cooldown, lockout, and alerting.

  • Try using OTP generated for User A/session A to verify User B/session B.

  • Try disabling 2FA without CSRF token and without password/re-auth confirmation.

  • Enable 2FA, reset password, then check whether login/dashboard is accessible without completing 2FA.

  • Use recovery/backup codes repeatedly, brute force them, or use codes from another session/user.

  • Check whether 2FA disable/manage page can be framed and abused via UI redress.

  • Submit empty/null/default/common OTP values.

  • After password login but before 2FA, open other tabs/API endpoints and check access.

  • Repeat/modify registration OTP requests and final submit requests to see if verification state can be bypassed.

Forgot Password - Request Phase 11 items

  • Submit valid and invalid email/phone and compare message, status code, response length, redirect, and timing.

  • Send many forgot-password requests for same account/IP and observe throttling/CAPTCHA/email flood protection.

  • Request reset links/OTPs repeatedly and check if target receives unlimited messages.

  • Submit duplicate or multiple email values and verify reset link is not sent to attacker address.

  • Add separators to email field and check if application sends to multiple recipients or mis-parses value.

  • Inject CRLF into email-related fields to add CC/BCC/Subject/body headers.

  • Submit incomplete email values and check if logic maps them to real accounts or bypasses validation.

  • Request forgot password and inspect API response for reset token/link/OTP/user ID.

  • Modify Host/X-Forwarded-Host during reset request and inspect reset email link domain.

  • Enter HTML/XSS-style email/name values and check whether reflected on page/email without encoding.

  • Submit malformed JSON, long strings, Unicode, invalid content type, or missing fields.

Password Reset - Token/OTP/Password Phase 23 items 
  • Generate a reset link/token and use it after its configured expiry window.

  • Use reset link once to change password, then use the same link again.

  • Request reset link A, then request reset link B, use B, then try A.

  • Request reset token, then log in and change password from account settings, then try old reset token.

  • Request reset token, change account email, then try old reset token.

  • Modify token, user_id, email, phone, or account_id in reset request.

  • Generate multiple reset tokens and compare for timestamp/user ID/email/name-based patterns.

  • Open reset page, click external/social links, and inspect outbound Referer header.

  • Try many invalid OTP codes and observe limits/lockout/cooldown.

  • Use OTP after expiry time.

  • Use valid OTP once, then try it again.

  • Use OTP sent to User A/session A with User B/session B by changing email/phone/user_id in request.

  • Call reset-password API without valid OTP/token or after deleting verification step parameters.

  • Capture successful reset request and resend it.

  • Send multiple parallel reset requests using the same valid token/OTP.
  • Set weak/common/passwords similar to username/email.

  • Try resetting to current or previous passwords.

  • Send different new password and confirm password values or tamper only one in API.

  • Modify redirect/returnUrl/next parameter after successful reset.

  • Submit reset/change-password request without CSRF token when cookie-based auth is used.

  • Reset password while active sessions exist in another browser/device, then test old sessions.

  • After reset/logout, use browser back button and cache/offline mode.
  • Inspect reset confirmation email/SMS/notification content.
Authorization Testing 0 items
Directory Traversal / File Include / Null Byte 4 items 
  • Modify file/path parameters to access parent directories or restricted files.

  • Try URL-encoded and double-encoded traversal.

  • Inject null byte in filename/path to bypass extension checks in vulnerable stacks.

  • If file/template/page parameters exist, try including local or remote resources in authorized scope.

Authorization Bypass and 403 Bypass 5 items 
  • Access restricted pages/APIs without login and as low-privilege user.

  • Try variations of restricted path with slashes, dots, semicolons, and encoded characters.

  • Try single/double encoded path traversal or dot encodings against restricted resources.

  • Add rewrite/original URL/IP override headers and check if backend routes to restricted resource.

  • Send restricted endpoint with different HTTP methods.

Privilege Escalation and IDOR 4 items

  • Change object/user IDs to access another user’s resources.

  • As normal user, modify role/permission/status fields or call admin APIs.

  • Modify IDs in GET/PUT/DELETE/download requests and verify object-level authorization.

  • Enumerate sequential IDs and check whether data is exposed.

Session Management Testing 0 items
Session and Cookie Controls 8 items 
  • Remove/expire/modify session token and call protected APIs/pages.

  • Inspect auth cookies for Secure, HttpOnly, SameSite, path, domain, expiry.

  • Capture session before login and verify session ID changes after login.

  • Search URL, response body, JS, localStorage/sessionStorage, logs for session IDs/tokens.

  • Remove/alter CSRF token on state-changing requests and test whether request succeeds.

  • Logout and replay old session/token requests.
  • Wait idle/absolute timeout and replay old session/API requests.

  • Mix sessions/tokens between users, roles, or incomplete workflows and check confusion.

Data Validation Testing 0 items
Injection and Input Validation 16 items

  • Inject script/HTML payloads in parameters/forms/headers and check immediate reflection/execution.

  • Store payloads in profile/comments/posts/uploads and view as another user.
  • Change request methods and verify validation/authz remains same.
  • Send duplicate/conflicting parameters in query/body/JSON/multipart.

  • Inject SQL metacharacters/operators in inputs and observe errors/timing/logic changes.

  • Inject LDAP filter characters in auth/search inputs.
  • Tamper filters/sort/search JSON used by ORM queries.
  • Submit XML with entities/doctype if XML is accepted.

  • Inject SSI directives where server-side includes may process content.

  • Inject XPath logic in XML-backed search/auth inputs.

  • Inject CRLF or mail command payloads in email-related fields.

  • Inject language/template expressions where dynamic evaluation may occur.

  • Inject OS command separators in parameters used by server commands.

  • Send oversized input values and malformed data to check crashes/errors.

  • Store payloads in one feature and trigger them later in another workflow.

  • Test CRLF header injection and safe request smuggling detection in authorized environments.

Cryptography Testing 0 items
Transport and Crypto Controls 3 items

  • Scan TLS versions, ciphers, certificate chain, hostname, expiry, and weak protocols.

  • Check whether credentials, tokens, OTPs, or PII are sent over HTTP or loggable URLs.

Business Logic Testing 0 items
Core Business Logic 7 items

  • Submit invalid business values such as negative amount, excessive quantity, invalid state, or unauthorized status.

  • Modify hidden/client-controlled fields in requests and replay actions.
  • Remove/tamper checksum/signature/hidden fields and check if request is accepted.
  • Send parallel requests for one-time/limited actions.
  • Repeat limited functions rapidly and across sessions/IPs.

  • Call later-step APIs directly without completing earlier steps.

  • Automate spam/mass actions/scraping/fake registration/excessive requests.
File Upload Functionality (WSTG-BUSL-08 / WSTG-BUSL-09) 19 items

  • Upload disallowed file types.

  • Change Content-Type to allowed value while body contains disallowed content.

  • Add valid image magic bytes to malicious/non-image content.
  • Upload SVG containing script/event handlers/external references.

  • Upload same filename multiple times or same path from different users.

  • Check whether risky files open inline or forced download.

  • Upload/download images/docs and inspect metadata.
  • Upload invalid/corrupt files and inspect errors.

Design and Abuse Case Testing 4 items

  • Check critical flows for missing rate limits, missing re-authentication, missing approval, or missing server-side validation.

Integrity and Supply Chain 5 items 
  • Identify outdated/vulnerable libraries and missing integrity controls.

  • Look for exposed build artifacts, pipeline files, logs, secrets, or deployment configs.

  • Decode and tamper JWT claims/header/signature; test alg=none/weak algorithms/expired tokens.

Security Logging & Monitoring Testing (A09) 0 items
Logging, Monitoring and Audit Trail 5 items

  • Perform login failures, password reset, privilege changes, admin actions, and check logs if available.

  • Verify logs do not contain passwords, OTPs, tokens, session IDs, or PII unnecessarily.

  • Inject CRLF/special characters into logged fields and inspect log formatting if accessible.

  • Verify critical actions record who, what, when, where, and before/after state.

Server-Side Request Forgery 6 items

  • Find functions where server fetches user-supplied URLs.

  • Use controlled callback domain to confirm whether server makes outbound request.

  • Submit internal URLs in authorized test environment.

Additional Advanced Web/API Vulnerabilities 0 items
Cache, Header, Redirect and Proxy Attacks 9 items

  • Find cacheable pages, add unkeyed headers/params, check reflection, then verify if poisoned response is served from cache.

  • As logged-in user, append fake static extension/path to private dynamic pages and test if cached publicly.

  • In authorized environments, use safe Burp Smuggler probes to test CL.TE/TE.CL/TE.TE desync.

  • Modify Host and forwarded host headers and check password reset links, redirects, asset URLs, cache, and routing.

  • Inject CRLF into inputs reflected in headers such as Location or Set-Cookie.

  • Decode and modify client-controlled cookies and replay requests.

  • Modify redirect parameters and test external/encoded bypasses.
  • Try headers that may be trusted by proxy/backend routing.
  • Inject CRLF into email fields to attempt CC/BCC/Subject/body injection.
Deserialization, Race and Prototype Pollution 4 items

  • Identify serialized data in cookies, hidden fields, API body, headers, uploads, or ViewState and modify low-risk values first.

  • Send parallel requests for one-time/limited actions and verify backend state.

  • Inject prototype keys into JSON/query inputs and check unexpected property/behavior changes.

  • View page source and JS files; search for tokens, API keys, secrets, internal endpoints, and source maps.

Additional Non-API-Top-10 VAPT Coverage Added 0 items

This section adds the remaining Web/API checks that were not part of the dedicated OWASP API Top 10 checklist. Items are written as direct test actions for pentesters and bug bounty hunters.

Additional Data Validation Checks 0 items
NoSQL Injection 3 items

  • Identify inputs used by NoSQL-backed features such as login, search, filters, profile lookup, or document lookup. Replace normal values with NoSQL operators and observe whether authentication, filtering, or authorization changes.

  • Send safe boolean-style NoSQL payloads and compare response length, result count, status code, and timing.

  • Change expected string/number fields into arrays or objects and check whether validation is bypassed.

Server-Side Template Injection (SSTI) 3 items

  • Find inputs rendered in server-generated pages, emails, PDFs, invoices, templates, or notification content. Submit basic template expressions and check whether they are evaluated.

  • Submit template expressions in profile name, invoice fields, report titles, email templates, or content generation fields and check generated output.

  • Send malformed template syntax and inspect errors for template engine names or stack traces.

CSV / Formula Injection 2 items

  • Enter formula-like payloads in fields that may be exported, such as name, email, comment, ticket title, address, or report data. Export CSV/XLSX and open safely in a controlled environment.

  • If the app imports CSV and later exports/reports it, upload formula payloads and verify they are sanitized during export/display.

Mass Assignment / Overposting 3 items

  • Add unauthorized fields to create/update requests and check whether backend accepts them.

  • Add fields that affect plan, price, wallet, limits, approval, ownership, or verification status.

  • Inject hidden fields inside nested JSON objects and arrays.

Additional Authentication, OAuth, SSO and JWT Checks 0 items
OAuth / SSO Misconfiguration 6 items

  • During OAuth/SSO login, modify redirect_uri/callback URL and verify whether tokens/codes can be sent to an untrusted domain.

  • Start OAuth login and check whether state exists, is random, tied to session, and validated. Remove/change/reuse it.

  • For mobile/SPA/public clients, check whether authorization code flow requires valid code_verifier matching code_challenge.

  • Use the same OAuth authorization code more than once.

  • Check whether access tokens, ID tokens, or auth codes appear in URL fragments/query parameters and leak to third-party links.

  • If ID tokens are used, modify issuer/audience/expiry/signature and check whether login is accepted.

Account Linking / Social Login Abuse 4 items

  • Try initiating account linking while logged in as one user and complete OAuth with another identity, or reuse linking URLs across sessions.

  • Check whether the app trusts email from OAuth provider even if email_verified is false or absent.

  • Try logging in with a provider account whose email differs from the existing local account or is controlled by attacker.

  • Combine SSO flow with redirect_uri/open redirect testing and check whether tokens/codes can be captured.

Detailed JWT Testing 6 items

  • Decode JWT, change header algorithm to none, remove signature, and replay in authorized test environment.

  • If HS256 is used and allowed by scope, test whether token can be cracked using weak/common secrets.

  • Modify claims such as role, user_id, tenant_id, scope, plan, or isAdmin and replay token.

  • Replay expired token or remove/change exp claim.
  • Change iss/aud claims or use token from another environment/app.

  • Modify JWT header key identifiers/remote key URLs and check whether server fetches or trusts attacker keys.

Rate Limit Bypass Techniques 4 items

  • Repeat rate-limited requests while changing spoofable IP headers.

  • Change case, spacing, encoding, plus addressing, or phone number format while attempting rate-limited actions.

  • Use alternate endpoints, API versions, trailing slashes, or mobile endpoints for the same action.

  • Change User-Agent, device ID, cookies, or create new sessions and repeat the same action.

Additional API-Connected Feature Checks 0 items
GraphQL Security 5 items

  • Identify /graphql or similar endpoint and verify whether it is intended to be public.

  • Send introspection query and check whether schema, types, queries, mutations, and fields are exposed.

  • Query sensitive fields not visible in UI or belonging to another user.

  • Send deeply nested or expensive queries and observe latency/rate limits.

  • Send multiple queries/mutations in one batch to bypass rate limits or perform mass actions.

Advanced CORS Testing 5 items

  • Send requests with arbitrary Origin header and check whether it is reflected in Access-Control-Allow-Origin.

  • Send Origin header with cookies/session and check Access-Control-Allow-Credentials behavior.

  • Send Origin: null and verify whether sensitive APIs allow it.

  • Try attacker-controlled domains that visually match trusted domains.

  • Send OPTIONS preflight and check whether dangerous methods/headers are allowed unnecessarily.

Webhook Security 4 items

  • If users can configure webhook URLs, try internal/private URLs and unsupported protocols.

  • Send a fake webhook request without signature, with invalid signature, and with modified body.

  • Replay a valid webhook request multiple times.

  • Check frontend, logs, API responses, and docs for webhook signing secrets or callback tokens.

Download / Export Functionality 5 items

  • Change file/report/export ID in download URL/API and check whether another user’s file can be accessed.

  • Modify filename/path parameters to access server files or other directories.
  • Use filters/pagination/body changes to export more data than visible/authorized.

  • Reuse signed download URLs after expiry/logout/password reset or from another account.

  • Download reports/exports and check for PII, internal IDs, tokens, hidden columns, or debug fields.

Sensitive Data Exposure in API Responses 4 items

  • Compare UI-visible fields with raw API response and identify extra PII/internal/security fields.

  • Call same endpoint with low/high privilege users and compare response data.

  • Trigger errors and inspect response for stack, SQL, internal service names, trace IDs with sensitive context.

  • Verify sensitive API responses are not cached by browser/CDN/proxy and not stored in local/session storage unnecessarily.

Additional Client, Headers and Supply Chain Checks 0 items
Security Headers Detailed Review 6 items 
  • Review CSP header for missing/weak directives and unsafe script sources.

  • Verify X-Frame-Options or CSP frame-ancestors on sensitive pages.

  • Verify X-Content-Type-Options header and dangerous content rendering behavior.

  • Click external links from sensitive pages and check whether tokens/paths leak in Referer.

  • Review whether browser features like camera, microphone, geolocation, clipboard are restricted.

  • Verify session cookies include Secure, HttpOnly, SameSite, and narrow Path/Domain.

Dependency / Frontend Script Integrity 4 items

  • List all external scripts loaded by the frontend and verify they are trusted, necessary, and not loaded over HTTP.
